# Fix: rncli.h file not found Error

## Problem
```
fatal error: 'rncli.h' file not found
#include <rncli.h>
```

## Solution 1: Clean Build और Regenerate (Recommended)

### Step 1: Complete Clean
```bash
# Project root से
cd android
./gradlew clean
rm -rf app/build
rm -rf .gradle
rm -rf app/.cxx
cd ..
```

### Step 2: Node Modules और Cache Clean
```bash
# Project root से
rm -rf node_modules
npm install
```

### Step 3: Expo Prebuild (Native files regenerate करने के लिए)
```bash
npx expo prebuild --clean
```

### Step 4: Build करें
```bash
cd android
./gradlew bundleRelease
```

---

## Solution 2: Expo Run Command Use करें (Easier)

Expo का command use करें जो automatically native files generate करता है:

```bash
# Project root से
npx expo run:android --variant release
```

यह command:
- ✅ Native files automatically generate करेगा
- ✅ rncli.h file create करेगा
- ✅ Release AAB build करेगा

**Note:** AAB file location: `android/app/build/outputs/bundle/release/app-release.aab`

---

## Solution 3: Manual Fix (Quick Workaround)

अगर ऊपर के solutions काम न करें, तो manually rncli.h create करें:

### Step 1: Directory Create करें
```bash
mkdir -p android/app/build/generated/rncli/src/main/jni
```

### Step 2: rncli.h File Create करें
```bash
# Windows PowerShell में
New-Item -Path "android\app\build\generated\rncli\src\main\jni\rncli.h" -ItemType File -Force

# या Linux/Mac में
touch android/app/build/generated/rncli/src/main/jni/rncli.h
```

### Step 3: Basic Content Add करें
File में यह content add करें:
```cpp
#pragma once

// React Native CLI generated header
// This file is auto-generated by React Native Gradle plugin
```

### Step 4: Build करें
```bash
cd android
./gradlew bundleRelease
```

---

## Solution 4: Disable New Architecture (Temporary Fix)

अगर New Architecture enable है और problem दे रहा है:

### `android/gradle.properties` में:
```properties
newArchEnabled=false
```

फिर clean build करें:
```bash
cd android
./gradlew clean
./gradlew bundleRelease
```

## Recommended Approach

**Best Solution:** Solution 2 use करें - `npx expo run:android --variant release`

यह सबसे reliable है क्योंकि:
- ✅ Expo automatically native files handle करता है
- ✅ rncli.h automatically generate होता है
- ✅ सभी dependencies properly link होती हैं
- ✅ EAS की जरूरत नहीं

## If Still Failing

अगर अभी भी error आए:

1. **Check React Native Version:**
   ```bash
   npm list react-native
   ```

2. **Update Dependencies:**
   ```bash
   npm update
   ```

3. **Check Android SDK:**
   - Android Studio में SDK Manager खोलें
   - Android SDK Platform 34 installed होना चाहिए
   - NDK 26.1.10909125 installed होना चाहिए

4. **Gradle Version Check:**
   ```bash
   cd android
   ./gradlew --version
   ```

आपके `build-release.ps1` और `build-release.sh` scripts में यह add करें:

**Before build, add:**
```bash
# Generate native files
npx expo prebuild --no-install
```

या directly use करें:
